### 坐标系（2026-04-27 双包实证定稿）

| 轴 | 方向 | 说明 |
|---|---|---|
| **X** | 前后 | 增大 = 向前（远离默认相机方向） |
| **Y** | 左右 | 增大 = 向右 |
| **Z** | 垂直（高度） | 增大 = 向上 |

**.ws 文件坐标换算规则**（2026-04-27 实证，挑战9验证 ✅）：
```
editor_X (cm) = ws.Position[0] × 30
editor_Y (cm) = ws.Position[2] × 30   ← 轴互换：ws[2] → editor Y
editor_Z (cm) = ws.Position[1] × 30   ← 轴互换：ws[1] → editor Z（高度）
```
> 即：1 ws单位 = 30 cm；ws 内部 Y/Z 轴与编辑器显示 Y/Z **互换**。

### 摄像机参数含义（实证）

| 参数 | 语义 | 单位 |
|------|------|------|
| `PointInPitch` | 俯仰角；**90 = 水平**，120 = 俯 30°，135 = 俯 45°，150 = 俯 60°，180 = 正俯视 | 度 |
| `PointInDirection` | 水平朝向；场景默认 -90 | 度 |
| `CameraFollow` 第 2 参数 | **DISTANCE**：相机在 **X 轴**（前后）相对角色的偏移量 | cm |
| `CameraFollow` 第 3 参数 | **OFFSET_Y**：相机在 **Y 轴**（左右）相对角色的偏移量 | cm |

| `CameraService.props.Current` | **仅影响编辑器预览**，运行时由 BlockScript 接管 | 字符串 |

> ⚠️ **公式**：`cam_editor = char_editor + (DISTANCE, OFFSET_Y, HEIGHT)`（三轴直接相加，cm）
> 俯45°居中条件：`HEIGHT = DISTANCE`，`OFFSET_Y = 0`

### 摄像机三种控制模式（2026-05-07 定稿）

> **⚠️ 关键区分**：三种模式坐标语义完全不同，agent 生成时必须选对，禁止混用。

| 模式 | 核心积木 | 坐标/参数语义 | 典型用途 |
|------|----------|--------------|---------|
| **跟随偏移** | `CameraFollow(目标, distance, offsetY, height)` | 相对目标角色的**偏移量**（cm）；目标移动，镜头跟着移动 | 主视角全程跟随角色，适合 OJ/关卡默认镜头 |
| **世界定位** | `GlideSecsToPosition3D(t, X, Y, Z)` / `GotoPosition3D(X,Y,Z)` | 摄像机的**世界绝对坐标**（cm）；与角色位置无关 | 运镜到固定机位、分镜切换 |
| **锁定追踪运镜** | `GlideSecsToPosition3D` ＋ 并行 `RepeatUntil(条件) { PointTowards(_, 目标) }` | 位置用世界坐标，朝向持续指向目标角色 | 镜头边移动边盯着角色，演出运镜 |

#### 模式三：锁定追踪运镜代码结构（来自视角参考包实证）

两个 `WhenGameStarts` 片段**并行执行**：

```
[片段A] WhenGameStarts          ← 控制镜头位移
  GlideSecsToPosition3D(t, X, Y, Z)   ← 滑动到世界坐标（cm），t 秒内完成
  （可连续多个 GlideSecsToPosition3D 做多段运镜）

[片段B] WhenGameStarts          ← 控制镜头朝向（与片段A并行）
  RepeatUntil(<结束条件>)
    PointTowards(_, 目标角色)   ← 每帧更新朝向，持续盯着目标
```

- `RepeatUntil` 的条件控制追踪持续时长，可以是：
  - 变量 `IsTrue(Variable(结束标志))` —— 收到消息后停止
  - `WaitSeconds(N)` 包在外层做总时长控制（见下方示例）
- 片段A 结束（Glide 完成）后镜头停在目标位，片段B 的追踪仍继续，直到条件满足

**典型演出用法**（t=3s 运镜 + 追踪）：

```
[片段A]
  GlideSecsToPosition3D(3, 2000, 1500, 2000)

[片段B]
  RepeatUntil(IsTrue(Variable(镜头结束)))
    PointTowards(_, 队长)
```

#### 模式对比速查

| 问题 | 用哪个模式 |
|------|-----------|
| 关卡全程跟着主角，保持固定俯仰角 | **跟随偏移** → `CameraFollow` |
| 分镜切到某个固定机位，不动 | **世界定位** → `GotoPosition3D` |
| 镜头推进同时一直对着角色（演出） | **锁定追踪运镜** → `GlideSecsToPosition3D` + `RepeatUntil+PointTowards` 并行 |
| 切镜头，镜头从A位滑到B位再停 | **世界定位** → 连续多个 `GlideSecsToPosition3D` |
| 开场是否需要运镜？ | 有剧情/人物登场 → **加运镜**；纯 OJ 练习无剧情 → **直接 CameraFollow，无需运镜** |

## 零、字段单位对照表（2026-04-23 三次复核，权威最终版）

> **2026-04-27 复核结论（第四次，坐标系+单位双重修正）**：
>
> - **ws 坐标单位**：`1 ws单位 = 30 cm`（不是 100cm/1m）。挑战9包实证：ws[0]=25.19 → editor X=757 ✓（×30）
> - **ws 轴顺序**：`ws[0]→edX, ws[1]→edZ(高度), ws[2]→edY(左右)`，ws 内部 Y/Z 与编辑器 Y/Z **互换**
> - **编辑器显示轴**：X=前后，Y=左右，Z=垂直（高度）
>
> **旧结论（已废弃）**："JSON 存米，UI × 100 = cm" —— 这是对 GotoPosition3D / MeshPart 等游戏逻辑坐标的描述，与 Character.props.Position 的 ws 存储格式不同。
>
> **权威硬规则（最终）**：
>
> - **ws Position 字段**：ws单位，× 30 → cm；轴：ws[0]→X，ws[1]→Z(高)，ws[2]→Y(横)
> - **GotoPosition3D / GlideSecsToPosition3D 坐标参数**：米（m），× 30 → cm，轴与编辑器一致（同 ws Position）
> - **CameraFollow 三参数**：cm，直接相对角色偏移，无需换算

| 字段 / 积木参数 | 单位（存储） | UI 显示 | 实证依据 |
|---|---|---|---|
| `Character.props.Position` / `MeshPart.props.Position` | **ws单位（= 1/30 m）** | 厘米（= ws值 × 30）；轴：ws[0]→edX, ws[1]→edZ(高), ws[2]→edY(横) | 10+ 参考包 Y=0.27（=8.1cm 地面）；三角对白 `["2","0.27","-1"]` |
| `Character.props.Size` / `MeshPart.props.Size` | **米（m）** | 米 | 小核桃 Size≈[1, 2, 1] = 身高 2m |
| `Character.props.Scale` | **倍数** | 倍数 | 1.0 = 原始大小 |
| `scene.props.BoundsCenter` / `BoundsSize` | **米（m）** | 米 | 标准 `[16, 7.67, 12]` |
| `GotoPosition3D(x, y, z)` | **ws单位** | 厘米（ws值 × 30） | "修改2" `Goto(0, 0.25, 0)` 与 Position 同量；14-1 `Goto(-146, -55, 8)` 与 MeshPart Position 同量纲 |
| `GlideSecsToPosition3D(sec, x, y, z)` 坐标 | **ws单位** | 厘米（ws值 × 30） | 同上 |
| `MoveSteps(n)` | **米（m）** | 厘米 | `MoveSteps("1")` = 走 1 米 |
| `CameraFollow(target, distance, offsetY, height)` 的 distance/offsetY/height | **厘米（cm），相对目标角色坐标的偏移量** | 厘米 | 实证：`offset = 摄像机编辑器坐标 − 角色坐标`；正视-中 `(887,636,350)` 验证通过（2026-04-27）|
| `SetCameraFOV` | 度 | 度 | — |
| `PointInDirection` / `PointInPitch` | 度 | 度 | — |
| `WaitSeconds` / `SaySeconds` | 秒 | 秒 | — |
| `ChangeSize(delta)` / `SetSize(size)` | **百分比**（100 = 原始） | 百分比 | `SetSize("100")` = 维持原大小 |

### 记忆口诀（最终版）

- **JSON 里 99% 都是米**：Position / Size / Bounds / Goto / Glide / MoveSteps。
- **摄像机 `CameraFollow` 的 distance/offsetY/height 是唯一的异类：厘米，且是相对角色坐标的偏移量**（历史遗留单位）。
- **编辑器 UI 的"厘米"只是显示层**：设计师输 `100` UI 是 1m，JSON 就存 `"1"`。我们写代码直接往 JSON 塞值时，必须塞米值（1m 写 `"1"`，不是 `"100"`）。
- **`ChangeSize/SetSize` 是百分比，不是长度**。

### 典型错误（2026-04-23 三角对白关卡事故复盘，再次修订）

```python
# ❌ 错误一：我在改规则时把单位改成了厘米，导致"正确写法"变成：
Character.props.Position = ["100", "27", "0"]    # → 解读为 100m / 27m / 0m，角色飞到 100 米外 27 米高

# ❌ 错误二（v1/v2/v3 真正原因猜测）：不是单位问题，而是布局问题——三人 Z 相近、X 又和 control 重合
Character.props.Position = ["1", "0.27", "0"]    # 队长
MeshPart(control).Position = ["1.5", "0.27", "0"] # control 在队长前方 0.5m → 视觉上贴在队长身上 = "重叠"

# ✅ 正确：JSON 用米值；让人物和 control 明显错开
Character.props.Position = ["2", "0.27", "-1"]   # 小核桃（前排左）
Character.props.Position = ["1", "0.27", "0"]    # 队长（后排中）
Character.props.Position = ["2", "0.27", "1"]    # 展喵（前排右）
MeshPart(control).Position = ["1.5", "0.27", "0"]  # 重心
GotoPosition3D("2", "0.27", "-1")                # 米值（三角对白参考包同款）

# ✅ 摄像机是唯一的厘米（历史遗留）：
CameraFollow("control", "640", "0", "500")       # 距离 640cm = 6.4m，高度 500cm = 5m
```

### 速诊（设计师报"角色重叠"时先查这里）

`verify_blocks.py` 不检查单位/布局。经验规则：

- **人物 Position 只差 <0.5m**（X、Z 都差不到 0.5）→ 会视觉重叠。人形角色最小间距建议 **≥ 1m**。
- **人物 Position 和 control（MeshPart）只差 <0.5m** → 人物会"站在仪器上"。固定做法：control 放在人群重心的前方 0.5m，并把 control 的 `Visible` 设 false。
- 若 X 或 Z 值超过 `±500`（米）→ 超出单屏视野，摄像机拍不到。
- 若谁写了 `Position=["100","27","0"]`（3 位数米值）→ 99% 是早先错把厘米当米，需要 ÷ 100。

### 场景锚点（万能模板，不会错）

- 地面 Y = **0.27**（米）——永远别写别的值，除非是空中场景
- 人形角色 Scale = 1.0，Size 参考角色表
- 单屏三人三角摆位（米）：
  - `(2, 0.27, -1)` / `(1, 0.27, 0)` / `(2, 0.27, 1)`（抗寒跑操 / 三角对白同款）
- control 放人群重心稍前：如 `(1.5, 0.27, 0)`，Visible=false

### 典型布局示例（Position 单位：米）

```
单排 5 个物件（间距 2m，居中）：
  Z 坐标: -4, -2, 0, 2, 4   → control Z = 0

单排 5 个物件（偏右布局，Z 从 0 开始）：
  Z 坐标: 0, 2, 4, 6, 8     → control Z = 4

双排布局（X 和 Z 都不同）：
  第1排: (0, 0.27, -2), (0, 0.27, 0), (0, 0.27, 2)
  第2排: (2, 0.27, -2), (2, 0.27, 0), (2, 0.27, 2)
  → control: (1, 0.27, 0)，使用预设C
```

### 摆放黄金法则（默认室内小场景）

- 单排物件沿 **Z 轴**排列，间距 **1.5~2 米**，整体居中在 Z=0 附近
- control 放在所有物件的 Z 轴均值处（物件对称则 Z=0）；`Visible` 必须 `false`
- 人形角色互相间距 **≥ 1m**，否则会视觉重叠
- 人形角色和 control 的距离 **≥ 0.5m**，否则会"站在仪器上"
- 物件 X 坐标不要超过 4（太深）或小于 -1（太近/出画面）

> ⚠️ 以上范围**只针对默认室内小场景**（28746 教室 16×12m）。非标准场景（室外/大地图/多层）必须用 `navmesh.md` 的验证工具。

### 角色朝向（PointInDirection 参考值）

- 不设置时，角色默认**面朝 X 轴正方向**（朝摄像机方向）
