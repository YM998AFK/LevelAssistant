# 新建关卡主 Agent · 执行流程

---

## §1 Step 1 · 一次性收集单

Agent 接到新建任务后，**立即**发出以下收集单（整块 markdown，不分多次提问）：

```markdown
## 关卡收集单（一次填完即可，非必填可留空/打"默认"）

| # | 字段 | 必填 | 预置选项 / 默认 | 你的填写 |
|---|------|------|----------------|---------|
| 1 | **主角** | ○ | 默认（乌拉呼星际服 Scale 0.6）/ 我给名字 / 等剧情推荐 / 没有主角 | |
| 2 | **运行模式** | ✅ | A 云编译（演示）/ B OJ（学生 cin）| |
| 3 | **展示代码** | 视模式 | 粘贴完整 C++（OJ 必有 cin/cout；云编译可无）| |
| 4 | **输入档位** | 视模式 | 云编译→几组输入+具体值 / OJ→支持哪几档+超档桥段 | |
| 5 | **视角** | ○ | 俯视角 45°（默认）/ 正俯视 / 其他；物件数 ≤4 / 5-8 / ≥9 | |
| 6 | **关卡流程** | ✅ | 一段自然语言：开场→中间→结算 | |
| 7 | **特别关注** | ○ | 如"箱子带序号"、"角色必须跑动"等 | |
| 8 | **场景** | ○ | 默认室内(28746) / 名称 / AssetId | |
| 9 | **BGM** | ○ | 冒险/治愈/紧张/轻松休闲(默认 28985)/不要 | |
| 10 | **核心物件** | ○ | 默认(宝箱/绿灯/零件/烟花) / 自由输入 | |

说明：○ = 可留空，agent 按默认补；✅ = 必填。
字段 1 选"等剧情推荐"时，资源搜索员会在收到字段 6 后做 Top3 推荐。
```

### 收到填写后的处理

1. **逐字段推导**：
   - 字段 3 代码 → 解析 cin/cout → 确定输入类型/个数 + 判题规则
   - 字段 2=B → 按字段 3+6 初判模板族（F.tree / F.sundial / F.beast 等）
   - 字段 5 视角 → 不自行推算 CameraFollow 参数，由生成员在其步骤3.7按 `../../_knowledge/positioning.md §3` 反推

2. **缺口回问（最多 1 轮）**：必填字段为空 / 有歧义 → 集中一次性问，不轮番追问

3. 进入 Step 2

---

## §2 Step 2 · 并行资源搜索 + 制作确认单

### 2.1 并行触发（同一消息批次）

**A. 扫包（有母本时）**：若收集单字段 `BASELINE_ZIP` 不为空，读取并执行扫包模块：

```
读 `../../_shared/scan_package.md`
变量：
  EXTRACTED_DIR_PATH = <母本 zip 解压后的 dest 绝对路径>
  WS_FILE_PATH       = <EXTRACTED_DIR_PATH 内的 .ws 绝对路径>
  SKILL_DIR          = <系统提示中的 Skill 目录绝对路径>
```

> 扫包目的是理解母本结构，语义转换中"改后变化"填"无（新建）"。无母本时跳过此步。

**B. 派遣资源搜索员**：读 `../../_shared/resource_agent.md`，填写任务后派遣：
- 任务 A：角色推荐（若字段 1 = "等剧情推荐"）
- 任务 B：物件选取（按字段 10）
- 任务 C：动画校验（若字段 1 已给定角色名，需核验该角色支持哪些动画）
- 任务 D：场景/BGM（按字段 8/9）

**B. 主 Agent 同时推导**：
- OJ 骨架：从代码解析 cin_cut 结构 / 变量 / 消息
- 分镜脚本：从字段 6 展开成逐步描述

### 2.2 汇总 → 制作确认单

等资源搜索员返回后，合并信息输出确认单：

```
## 生成前确认单

关卡名  : <name>
主角    : <角色名> (AssetId=<id>，Scale=<值>)（若推荐则附 Top3，让设计师挑）
场景    : <名> (AssetId=<id>)
BGM     : <名称 / AssetId>
物件    : <列表，每项含 AssetId + 来源（Top1/备选）>
剧情轴  : 1) ... 2) ... 3) ...
判题    : <OJ 简化模式 / 教师端 / 自定义>
视角    : <正视 / 俯视 / 跟随 ...>（具体 cm 参数由生成员在步骤3.7按 positioning.md §3 反推计算）
资源缺口: <有则列 / 无则写"无">
产出路径: output/new/<name>.zip

确认生成? (y/n)
```

> M7 强制卡点：未收到设计师明确 OK，**禁止**派遣生成员。

### 2.3 资源搜索员质检

收到资源搜索员返回，确认：
- 每项资源都有 AssetId 证据？
- 未找到的需求已明确标注"缺口"？

## §3 Step 3 · 派遣生成员 + 质检 + 交付

### 如何填写 creator_agent.md 的变量

读 `agents/creator_agent.md`，将以下变量替换后，整体作为子 Agent prompt 发出：

| 变量 | 填写内容 |
|------|---------|
| `${LEVEL_NAME}` | 关卡名 |
| `${BASELINE_ZIP}` | 母本 zip 路径（无母本则留空） |
| `${RESOURCE_LIST}` | 确认单里所有 AssetId + Position + Scale |
| `${STORYBOARD}` | 分镜脚本（逐步描述） |
| `${OJ_PARAMS}` | 模板族 + 变量列表 + 档位定义 + 判题规则（OJ 关卡必填；云编译留空） |
| `${CAMERA_PARAMS}` | 视角参数（生成员在步骤3.7按 positioning.md §3 反推计算） |

### 生成员返回后的质检四问

收到生成员交付报告，以下全部 YES 才进入交付：

1. validate_workspace error_count == 0？
2. verify_gates.py gate1/2/3/4/5 均 PASS（以脚本实际输出为准）？
3. 审查员返回 `总体：PASS`？
4. 包路径在 `output/new/` 下，文件名正确？

任一 NO → 告知生成员重做（新开 Task，不 resume）。

### 交付消息格式

```
## 交付

包文件  : output/new/<name>.zip (<大小> KB)
关键效果: <引用分镜1~3句话>
资源清单: 主角 AssetId=<id> / 场景 AssetId=<id>
校验结果: verify_gates PASS(5/5)  validate error_count=0
审查员  : 总体 PASS（M3: <动画结论> / N7: <移动结论> / 观赏性: <结论>）

MUST/NEVER 自检：M1-8 ✅ / N1-12 ✅
（🤖 verify_gates PASS / 🤖 MCP validate PASS / 🔍 审查员 PASS / 👀 人工已确认 M4/M7）
```

审查员报告里 M3/N6/N7 以及观赏性的具体结论，必须在交付消息里转述，不能只写"审查员 PASS"。

**输出精简上限**：交付摘要 ≤15 行；确认单 ≤15 行；每条追问说明 ≤3 行。
