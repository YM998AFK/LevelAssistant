# OJ 标准变量 & 消息包 v1

> **v1 更新日期**：2026-04-23
> **语料扩展**：从 6 包 → **61 包**（`参考-extracted/` 全量扫描，不含 `_mine/`）
> **来源工具链**：`scripts/ref_scan.py` → `ref_ingest.py` → `ref_distill.py` → `suggestions.md`
> **用途**：新建 OJ 关卡时，`scene.props2` 和 `#EVENT` 两个表必须先预置到位；此文件是 checklist。

---

## 0. 分级说明

共现率 = "该变量/消息在 61 个真参考包中的出现包数占比"。

| 级别 | 共现率阈值 | 语义 | 新建包时处理 |
|---|---|---|---|
| **L0 骨架必需** | ≥90% | OJ 脚手架无法运行的基础 | **必须**在 props2 / #EVENT 里预置 |
| **L1 高频推荐** | 75%~90% | 有判题演出的包基本都有 | 做判题演出就预置 |
| **L2 族专属** | 30%~75% 且集中在某族 | 按选中的模板族一次性全带 | 按 `cinematics.md` 族的依赖表 |
| **L3 包独有** | <30% | 题目专属 | 按需新增，勿反向污染骨架 |

> 共现率引自 `参考-extracted/suggestions.md` §2；本文件是**审核后的结论**。

## 1. L0 · 骨架必需变量（14 个，v1 新增 2 个）

新建 OJ 关卡，`scene.props2` 必须至少包含：

| 变量名 | 类型 | 覆盖率 | 作用 |
|---|---|---|---|
| `variable` | （系统） | 100% | Pangu3D 变量命名空间锚 |
| `#EVENT` | `{value: [...]}` | 100% | 本包所有 Broadcast 消息的声明清单 |
| `*OJ-输入信息` | string | 95% | 云端回填：学生 cin 的原始输入串 |
| `*OJ-执行结果` | string | 95% | 云端回填：学生 cout 的原始输出串 |
| `cmd` | "init"/"cintest"/"couttest" | 95% | 启动时的运行阶段 |
| `err_msg` | string | **93%** | 失败分支写入的错误文本（前端读取展示）**v1 新增 L0** |
| `state` | string | **93%** | 关卡运行状态文案 **v1 新增 L0** |
| `cin_cut` | list[str] | 91% | 输入切片数组；**演示模式下也用于预载演示输入**：WhenGameStarts 里用 `ListAdd` 序列依次填入演示用的 cin 值，代替从玩家输入解析的流程 |
| `cout_cut` | list[str] | 86% | 输出切片数组 |
| `n` | int-string | 86% | 通用数字寄存器（多为输入主值 / 循环上限）|
| `space-flag` | "0"/"1" | 86% | 切分空格/制表符标志 |
| `输入元素` | list[str] | 86% | 解析后的输入元素列表 |
| `输出元素` | list[str] | 86% | 解析后的输出元素列表 |
| `*OJ-Judge` | "0"/"1" | 83% | 云端回填：判题结果；**WS 设计者也可在 WhenGameStarts 手动置 `'1'`** 触发演出演示（`*OJ-Judge='1'` 是进入演示分支的开关，不仅限于云端回填）|

> 以 `*OJ-` 开头的三个变量是**云端-关卡契约**，改名等于解约。
> `err_msg` 和 `state` v1 晋升：93% 覆盖率已超过 90% 阈值，与 cin_cut / n 等处于同一梯队。

## 2. L0 · 骨架必需消息（4 个）+ L0.5 判题消息（2 个）

`scene.props2.#EVENT.value` 必须至少包含：

### L0（>90% 共现）

| 消息名 | 覆盖率 | 发送者（典型） | 接收者 handler | 作用 |
|---|---|---|---|---|
| `CMD_NewMessage` | 100% | 系统内置 | 系统内置 | 命令新消息低层 hook |
| `传递失败` | 95% | 错误分支 | `control.WhenReceiveMessage("传递失败")` | 统一失败出口，关卡失败 UI/音效 |
| `初始化` | 95% | `WhenStartup` / C-2 init 分支 | 多节点 handler | 初始化所有 UI / 变量 |
| `传递成功` | 93% | 正确分支末尾 | `control.WhenReceiveMessage("传递成功")` | 统一成功出口，跳下一关 |

### L0.5（80%~90%，v1 细化）

| 消息名 | 覆盖率 | 说明 |
|---|---|---|
| `展示关卡效果` | **80%** | 结算演出消息，20 包命中"传递失败→展示关卡效果→传递成功×2" 4 步结算链，是 C-4 标准姿势 **v1 新设 L0.5** |
| `运行` | 78% | C-1/C-2 的主分派消息；部分老包用"启动"等变体 |
| `judge` | 78% | C-2 `cmd==init` 分支触发；部分老包用"进入判题"等变体 |
| `cin判断` | 77% | C-3 的 cin 合法性入口 |
| `cout判断` | 77% | C-4 的判题分岔入口 |

> **v1 修订**：`运行` / `judge` 从原 L0 降为 L0.5，因为新增的部分族（F.label / F.supplies 等）用变体名。但**新建 OJ 仍强制用标准名**，变体仅在改老包时保留。

## 5. L3 · 反例与非标准（低16-2 练习2）· 不要照抄

此包**没有** `cin判断` / `cout判断` handler，改用自研三件套：
- `判断是否输入` 替代 cin 合法性
- `判断输入数据范围` 替代 cout 前的范围检查
- `能否运行` 由 `判断输入数据范围` 根据结果决定是否广播 `运行`

独有变量（实时过程演示相关）：
`x, y, z, x速度, y速度, z速度, 分割数据n, 分割数据n1, 当前秒数, 当前过程的i, 是否结束循环, 正确效果？, 答案正确, 输入内容, 输入数据的个数, 过程运行次数`

**警告**：该架构**没有 `cin判断` / `cout判断` handler**，如以它为母本做新题，会引入额外复杂度。新 OJ 题应优先选 F.tree / F.sundial / F.beast / F.supplies / F.forge / F.label / F.fire / F.char / F.traverse。

## 6. 新建 OJ 关卡的变量/消息 checklist（v1 扩展）

按顺序打勾：

### 6.1 骨架层（所有 OJ 必需）
- [ ] `props2` 已包含 §1 列出的 **14 个 L0 变量**（v1 含 err_msg / state）
- [ ] `#EVENT.value` 已包含 §2 的 **4 个 L0 消息** + 5 个 L0.5 消息
- [ ] `control` 节点已挂 L0 + L0.5 的 handler：`运行 / judge / cin判断 / cout判断 / 传递失败 / 传递成功 / 初始化 / 展示关卡效果`
- [ ] `control` 挂有 `WhenGameStarts` 做 C-1 启动握手
- [ ] `control` 挂有 `WhenStartup` 做 C-5 初始化

### 6.2 cin 判断层
- [ ] 选定 C-3 形态：`[ ] 最简（单值）` / `[ ] F.number 增强（多值 / 浮点 / 非数字）`
- [ ] 若选 F.number：已加入 4 个增强消息 + handler

### 6.3 cout 判题结算
- [ ] C-4 的"4 步结算链"已实现：`传递失败 → 展示关卡效果 → 传递成功 → 传递成功`

### 6.4 族层（按选定模板族全带，见 §4）
- [ ] 选定族：`[ ] F.tree / [ ] F.sundial / [ ] F.beast / [ ] F.supplies / [ ] F.forge / [ ] F.label / [ ] F.fire / [ ] F.char / [ ] F.traverse / [ ] 自创`
- [ ] 族内变量已全部加入 props2
- [ ] 族内消息已全部加入 #EVENT
- [ ] 族内 handler 已全部挂上（见 `cinematics.md` §13 依赖表）

### 6.5 题目层
- [ ] 本题独有变量已加入，并在 SetVar 前**显式初始化**（不要依赖隐式 0/""）
- [ ] 本题独有消息已加入 #EVENT 且有对应 handler
- [ ] 若"一包多练"，已通过 `cid` 分发（见 `cinematics.md` §4 S-3）
- [ ] 若"变长序列"（如 F.fire 的记录火堆），**用循环而不是线性展开**

### 6.6 禁忌
- [ ] **没有**随便裁剪 L0 的 14 个变量 / 4 个消息
- [ ] **没有**跨族混搭 Broadcast 序列
- [ ] **没有**把 `*OJ-输入信息 / *OJ-执行结果 / *OJ-Judge` 改名或当作普通变量赋值
