# 剧情员主 Agent · 执行流程

---

## §1 Step 1 · 接收剧本

设计师通常直接粘贴剧本文本（无需填表）。

> **裸触发处理**：若触发词消息中**未包含剧本文本**（仅说"试试/来一个/写个剧情"等），只做一件事：请设计师发送剧本原文。收到剧本后，再执行下方的一次性询问。

主 Agent 接收剧本后，**仅一次性询问以下两件事**（若能从剧本推断则跳过对应问题，不轮番追问）：

1. **ws 包路径**：是否有参考 ws 包？（提供角色初始站位和场景）
   - 若设计师提供了 ws 包，则该包为**站位参考包**：包内所有角色/物件的 Position 与 EulerAngles（朝向）即为最终初始站位，**不得在产出包中覆盖**。
2. **关卡名**：输出文件的名称？（若剧本首行有地图名则默认用地图名）

> **询问完成后**：收到设计师回复，即确认三项（剧本 + ws包路径/无 + 关卡名）齐全 → **立即读 `steps/workflow.md §2` 进入 Step 2，禁止在此消息中再读任何其他文件**。

---

## §2 Step 2 · 并行派遣

**同一消息批次触发以下内容**：

### A. 扫包（仅有 ws 包时）

```
run_python(
  script_rel = "<Skill目录>/scripts/_scan_level.py",
  args       = [WORKDIR_PATH, WS_FILE_PATH]
)
```

从扫描输出提取 `${SCAN_CHARACTER_TABLE}`：

```
| 角色名 | AssetId | Position(x,y,z) | PointInDirection(°) | Scale |
|--------|---------|----------------|---------------------|-------|
| ...    | ...     | ...            | ...                 | ...   |
```

**注意**：若 ws 包含已有 BlockScript，**只取场景树信息**（角色/物件位置），不复用旧 BlockScript——全部依据剧本重新生成。

### B. 派分镜解析员

读 `agents/storyboard_agent.md`，填入以下变量后整体作为子 Agent prompt 发出：

| 变量 | 填写内容 |
|------|---------|
| `${SCRIPT_TEXT}` | 剧本原文全文（设计师粘贴的完整文本） |
| `${SCAN_CHARACTER_TABLE}` | A 步扫包提取的角色表（无 ws 包则填"无"） |
| `${SKILLS_DIR}` | 系统提示中的 Skill 目录绝对路径 |

### C. 派资源搜索员

等分镜解析员返回后，取其【资源搜索任务清单】，读 `../../_shared/resource_agent.md`，填入 `${SEARCH_TASKS}` 后发出。

> **执行顺序**：A（扫包）与 B（分镜解析员）必须同批次并行触发。C（资源搜索员）依赖 B 的输出（资源搜索任务清单），B 返回后**立即**在同一消息批次发出 C，不得另开轮次等待。

### 返回质检

收到分镜解析员返回，确认：
1. 分镜表中每行都有分类标签和时长估算？
2. ❓ 待确认项已全部列出？
3. 资源搜索任务清单完整（任务A/C/D 均有内容）？

## §3 Step 3 · 分镜确认单 + M7 卡点

等分镜解析员 + 资源搜索员均返回后，合并信息输出确认单（**≤25 行**）：

```
【剧情演出确认单 v1】

场景  : <名称>  AssetId=<id>（缺口：未找到 → 建议<候选> / 请设计师提供）
BGM   : <名称>  AssetId=<id>

角色清单：
  <角色名>  AssetId=<id>  初始位置=(<x>,0.27,<z>)  朝向=<°>°  Scale=<值>
  <延迟入场角色>  AssetId=<id>  登场场次=S<N>  位置=<待规划>  朝向=❓

物件清单：
  <物件名>  AssetId=<id>  ×<N>  位置：(<坐标列表>)

分镜摘要（共 N 场次）：
  S0 全屏特效[Scene.PlayAnimation] → S2 粉尘+音效 → ...（摄像机行括注"不实现"）

对白框  : SetVar(bubbleText, "xxx") → BroadcastMessageAndWait("显示对白")
全屏特效: Scene节点.PlayAnimation(assetId)  ← 非角色特效，目标节点为 Scene

资源缺口（若有）：
  - <描述>：<候选 / 请设计师提供>
  - [图片]×N：请补充说明图中关键意图（特效位置/角色站位等）

产出路径：output/new/<关卡名>.zip

确认生成？（回复"确认/可以/OK"；或告知需要调整的地方）
```

**粒度规则**：
- ✅ 写：角色做了什么、对白内容、特效/音效类型、物件位置
- ❌ 不写：动画名、fragment 编号、MCP 工具名、技术实现细节

> **M7 强制卡点**：确认单发出后，必须等设计师明确回复，禁止提前派生成员。
>
> 若设计师修改了角色/位置/台词/资源 → 更新对应行 → 输出确认单 v2 → 再次等 OK。

## §4 Step 4 · 派生成员

### 判断分支

| 条件 | 派遣对象 | 读的文件 |
|------|---------|---------|
| 设计师提供了 ws 包（站位参考包） | 改造员（以参考包为母本，仅重写 BlockScript） | `../../level-modify/agents/modifier_agent.md` |
| 无 ws 包 | 生成员（从模板新建，按三层流程规划站位） | `../../level-new/agents/creator_agent.md` |

### 传入变量填写

读对应 agent 文件，将以下变量填入后整体作为子 Agent prompt 发出：

| 变量 | 填写内容 |
|------|---------|
| `${LEVEL_NAME}` | 关卡名（剧本元数据或设计师指定） |
| `${BASELINE_ZIP}` | ws 包路径（无则"无"）；有站位参考包时填参考包路径 |
| `${RESOURCE_LIST}` | 确认单中角色+物件+特效+音效的 AssetId 全列表（含 Position / Scale） |
| `${STORYBOARD}` | 分镜解析员返回的完整分镜表（含对白原文、时序、实现方向；镜头方向行标注"不实现"） |
| `${OJ_PARAMS}` | 留空（纯剧情演出，无 OJ 判题） |
| `${CAMERA_PARAMS}` | 留空（剧情员不处理摄像机） |
| `${REF_POSITION_TABLE}` | 有站位参考包时，填扫包所得的 `${SCAN_CHARACTER_TABLE}`；无参考包时留空 |

### 生成员返回后质检四问

收到生成员交付报告，以下全部 YES 才进入交付：

1. validate_workspace error_count == 0？
2. verify_gates gate1–5 全 PASS（以脚本实际输出为准）？
3. 审查员返回 `总体：PASS`？（重点关注 N6 间距 / N7 移动冲突 / 观赏性）
4. 包路径在 `output/new/` 或 `output/modify/` 下，文件名正确？

任一 NO → 告知生成员重做（**新开 Task，不 resume**），携带具体失败原因和审查员报告。

### 交付消息格式

```
## 交付

包文件  : output/new/<name>.zip (<大小> KB)
关键演出: <引用分镜摘要 1~3 句话>
资源清单: 主角 AssetId=<id> / 场景 AssetId=<id>
校验结果: verify_gates PASS(5/5)  validate error_count=0
审查员  : 总体 PASS（N6: <间距结论> / N7: <移动结论> / 观赏性: <结论>）
```
