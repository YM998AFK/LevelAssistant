## 0. 使用原则

1. **先判模板族，再挑桥段**。每个桥段属于某个"模板族"（见 §1），跨族拼接会缺 handler。
2. **桥段是字面量序列**。所有 Broadcast 用原字符串（含中文、空格），不要自作主张改名。
3. **桥段之间是串联**。全部用 `BroadcastMessageAndWait`，保持时序。
4. **遇到未登记的 Broadcast 序列**，先查 §10 噪声规则，不是噪声就当作新桥段候选进 `suggestions.md`。
5. **本文件的"已固化"桥段**需 ≥3 包共现；2 包的标记"候选"；1 包的暂不入库。
6. **若发现 `suggestions.md` 里 §1 Top 桥段未进本文件**，说明是新入库批次，人工评审后按本模板追加。

## 2. F.core · OJ 核心骨架（所有 OJ 关卡必需）

### C-1 `OJ 启动握手`（3 次运行消息分派）

**字面量序列（3 个 Broadcast，夹着 SetVar）**：
```
SetVar(cmd, "init")     → Broadcast("运行")
SetVar(cmd, "cintest")  → Broadcast("运行")
SetVar(cmd, "couttest") → Broadcast("运行")
```

**上下文**：`WhenGameStarts` → `WaitUntil(*OJ-输入信息 ≠ "" AND *OJ-执行结果 ≠ "")` 之后。

**依赖 handler**：`WhenReceiveMessage("运行")` 必须按 cmd 分派到 C-2。

**覆盖率**：37/61 包（61%，顶级 n-gram）。其余包用 init/cintest/couttest 的变体。

---

### C-2 `运行分派`（cmd 路由到判题三段）

**逻辑（if 分派，不是线性序列）**：
```
WhenReceiveMessage("运行"):
    if cmd == "init":
        <调 MyBlock 初始化>
        BroadcastMessageAndWait("收起UI")   # 各族消息名不同
        BroadcastMessageAndWait("judge")
    if cmd == "cintest":
        BroadcastMessageAndWait("cin判断")
    if cmd == "couttest":
        BroadcastMessageAndWait("cout判断")
```

**关键 handler**：`judge` / `cin判断` / `cout判断` 三者缺一演出就哑火。

**反例**：F.nonstd（低16-2 练习2）用 "判断是否输入 / 判断输入数据范围 / 能否运行" 替代，**不要照抄**。

---

### C-3 `cin 合法性校验`（输入不合法就走错误分支）

**最简形态**（单值题，如挑战2）：
```
WhenReceiveMessage("cin判断"):
    If IsLess(ListGetItemAt(1, cin_cut), 1):
        SetVar(err_msg, "读取的位数超出范围,请联系管理员")
        BroadcastMessageAndWait("传递失败")
```

**增强形态 F.number`（多值 / 浮点 / 非数字检查，7 包共用，v1 新增）**：
```
WhenReceiveMessage("cin判断"):
    BroadcastMessageAndWait("判断是否有小数")
    BroadcastMessageAndWait("判断是否为非数字")
    BroadcastMessageAndWait("判断输入数据的个数")
    # 某些族再加一层范围
    BroadcastMessageAndWait("判断输入数据大小")   # L14-3 练习14/15
    # 任一 handler 内若 fail 则：
    # SetVar(err_msg, "<具体原因>"); BroadcastMessageAndWait("传递失败")
```

**依赖 handler**（7 包内全有）：`判断是否有小数` / `判断是否为非数字` / `判断输入数据的个数`（可选 `判断输入数据大小`）

**来源**：L14-2-低-练习15/16/终极挑战、L14-3-低-练习14/15 共 7 包。

---

### C-4 `cout 判题分支头`（按 *OJ-Judge 分岔）

**字面量骨架**：
```
WhenReceiveMessage("cout判断"):
    If *OJ-Judge == "0":
        SetVar(err_msg, "<错误提示>")
        BroadcastMessageAndWait("传递失败")
    If *OJ-Judge == "1":
        <族特定正确演出，见 §3~§9>
        BroadcastMessageAndWait("展示关卡效果")
        BroadcastMessageAndWait("传递成功")
        BroadcastMessageAndWait("传递成功")   # 双发：音效 + 下一关
```

**v1 确立**：`传递失败 → 展示关卡效果 → 传递成功 → 传递成功` 这 4 步结算链 **20 包共有**，已固化为**标准姿势**。

**硬规则**：err_msg 由云端/前端读取展示，不要自己写 LabelBubble。

---

### C-5 `初始化广播`

单独 `WhenStartup`（进场）执行一次：
```
WhenStartup():
    SetVar(*OJ-输入信息, "")
    SetVar(*OJ-执行结果, "")
    SetVar(err_msg, "")
    SetVar(cmd, "")
    SetVar(state, "...")       # 族特定默认状态
    BroadcastMessageAndWait("收起UI")   # 族特定隐藏 UI 消息
```

## 3. F.tree · 成长树 / 计数机器人（挑战2、挑战3、15-2 关卡3）

### T-1 `启动演出 - 7 步`（族招牌，3 包共用，v1 验证）

**字面量序列（L7，3 包命中）**：
```
传递失败 → 初始化 → 准备 → 机器人左转 → 机器人发射 → 屏幕显示 → 计算
```

**依赖 handler**：`准备` / `机器人左转` / `机器人发射` / `屏幕显示` / `计算`
**调用点**：`WhenReceiveMessage("cout判断")` 正确分支开头。
**来源**：挑战2, 挑战3, 15-2 关卡3。

---

### T-2 `成长树展示 - 4 步`

**字面量序列**：
```
收起 → 机器人右转 → 生成成长树 → 收起成长树
```

**依赖 handler**：`收起` / `机器人右转` / `生成成长树` / `收起成长树`
**位置**：每次判题正确后 "又长了一截"。
**来源**：挑战2 × 1，挑战3 × 2，15-2 关卡3 × 2（循环演出里用两次）。

---

### T-3 `单次判题完整演出`（挑战2）
顺序严格：
```
收起 → 角色升级 → 讲解框幕布出现 → 讲解框幕布消失 → 单独计算动作
  → 判断 → 讲解框幕布完全消失 → 拉近镜头切换
  → (WaitSeconds 1) → 拉远镜头切换特效 → 传递成功
```
**适用**：单值 → 判一次 → 打印一次（挑战2 的 `x%5==0`）。

---

### T-4 `循环演出`（挑战3）
```
if 0 < cin_cut[1] < 11:
    <T-2>
    Repeat(cin_cut[1]):
        每次生长一个 → 筛查
        if 关卡i < n: Broadcast(+1)
        关卡i += 1
    <切场>
    Repeat(10):   # 固定 10 次满演示
        同上
    <收尾>
    Broadcast(传递成功)
```
**⚠️ 潜在坑**：`n > 10` 时演出完全哑火（既不失败也不成功），见 `pitfalls.md` P-1。

## 12. 桥段选择决策树（做关卡时用）

```
1. 是否 OJ 关卡？
   └─ 否 → 此文件不适用（另见云编译/对白类参考）

2. 判题演出风格？
   ├─ 开门 + 计数机器人 + 成长树          → F.tree（挑战2/3, 15-2 关卡3）
   ├─ 日晷 + 测算循环 + 长老验算           → F.sundial（练习5/7/9）
   ├─ 分镜切换 + 血量对打                  → F.beast（挑战9/10/12, ym, 隐藏练习）
   ├─ 物资箱清点 + 跑位演出                → F.supplies（14-3-6/7/9）
   ├─ 炮弹落下 + 凸显框 + 勾叉             → F.forge（14-4 系列）
   ├─ 标签出现 + 电池滑行                  → F.label（L14-2 练习16 系列）
   ├─ 起火 + 无人机 + 火堆坐标             → F.fire（L14-3 练习13/14/15）
   ├─ 字符逐个出现 + 变色                  → F.char（S低/S高 L13/L14 系列）
   ├─ 基础遍历 + 对白框                    → F.traverse（低L14-3 练习12 系列）
   └─ 其它 / 不确定                        → 主动问设计师，不要自创

3. 判题形态？
   ├─ 单次判题（if 一次）                  → T-1+T-3 / S-1+S-2 / ...
   ├─ 循环判题（for + if N 次）            → T-4 / FR-2 / 族内循环变体
   └─ 一包多练（cid 分发）                 → S-3 + 族内实现

4. cin 合法性？
   ├─ 单值校验（只检查下界）              → C-3 最简
   └─ 多值 / 浮点 / 非数字检查             → C-3 F.number 增强版（4 合 1 判断）

5. 骨架层（所有 OJ 都要）：
   └─ C-1 → C-2 → C-3 → C-4 → C-5 五件套
```

## 14. 噪声规则（扫描识别但不入库）

| 序列 | 为什么不是桥段 |
|---|---|
| `传递失败 × N` | 同一消息的重复广播，是"失败/重试"的强化，非独立桥段 |
| `运行 → 运行 → 运行` | C-1 的 3 连发被扁平化采样，真实语义是"三阶段启动"非线性桥段 |
| `初始化 → judge → cin判断 → cout判断` | C-2 的 if 分派结构扁平化，真实语义是"三选一" |
| `传递失败 → 运行 → 运行 → 运行` | 跨 fragment 拼接（C-5 收尾 + C-1 起点） |

> 增量扫描时遇到"序列里全是同一消息重复"或"跨 hat 拼接"，默认不入库。
